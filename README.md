# VibeCodeReporter

A pipeline for mining code from GitHub repositories to build datasets for AI code detection analysis. Extracts commit history, source code, diffs, and computes code metrics.

## Overview

This tool:
1. Finds GitHub repositories by domain/language
2. Builds a metadata dataset of repositories
3. Mines code from repositories (full source + diffs)
4. Computes code metrics (LOC, complexity, comment density)
5. Outputs per-repo CSV checkpoints that can be merged

## Installation

```bash
# Clone the repository
git clone <repo-url>
cd VibeCodeReporter

# Create conda environment (recommended)
conda create -n git-mining python=3.11
conda activate git-mining

# Install dependencies
pip install -r requirements.txt
```

### Dependencies
- `pydriller` - Git repository mining
- `pandas` - Data manipulation
- `tqdm` - Progress bars
- `radon` - Python code complexity metrics
- `requests` - HTTP requests for GitHub API
- `python-dotenv` - Environment variable management

## Usage (WIP)

### Step 1: Build Metadata Dataset

First, create a metadata CSV with repository information:

```bash
python src/2_build_meta_dataset.py
```

This reads JSON files from `data/` and creates `data/repos_dataset.csv`.

### Step 2: Mine Repositories

Run the main mining script:

```bash
python src/3_repo_mining.py
```

#### CLI Options

```bash
python src/3_repo_mining.py [OPTIONS]

Options:
  --metadata-csv PATH       Input metadata CSV (default: data/repos_dataset.csv)
  --output-dir PATH         Output directory (default: output)
  --checkpoint-dir PATH     Checkpoint directory (default: output/repo_checkpoints)

  # Date filters
  --since DATE              Only commits after this date (YYYY-MM-DD)
  --to DATE                 Only commits before this date (YYYY-MM-DD)

  # Repository filters
  --min-repo-commits N      Min commits for repo (default: from config)
  --min-branch-commits N    Min commits per branch (default: from config)
  --max-repo-size-mb N      Skip repos larger than N MB

  # Commit sampling
  --commit-skip N           Take every Nth commit (e.g., 10 = every 10th)
  --min-commit-interval-days N  Min days between commits

  # File filters
  --min-file-size N         Min file size in bytes (default: 500)
  --max-file-size N         Max file size in bytes (default: 100000)
  --min-lines N             Min lines of added code (default: 10)

  # Content filters
  --include-autogenerated   Include auto-generated files
  --include-vendored        Include vendored/third-party code
  --exclude-tests           Exclude test files

  # Parallelization
  --num-workers N           Threads for commit processing (default: from config)
  --max-cloned-repos N      Max repos to clone ahead (default: 10)

  # Subset selection
  --limit N                 Process only first N repos
  --domain DOMAIN           Filter by domain
  --language LANG           Filter by language
  --repo-ids ID [ID ...]    Process specific repo IDs
```

### Step 3: Merge Checkpoints

After mining, merge per-repo CSVs into a single dataset:

```bash
python src/4_merge_repo_datasets.py
```

Options:
```bash
--checkpoint-dir PATH   Directory with checkpoint CSVs
--output PATH           Output merged CSV path
--quiet                 Suppress progress output
```

## Configuration

All settings can be configured in `src/configs.py`:

### File Paths
| Option | Default | Description |
|--------|---------|-------------|
| `data_base_path` | `data/` | Base path for input data |
| `output_base_path` | `output/` | Base path for outputs |
| `clone_repo_to` | `data/temp/` | Temporary clone directory |
| `checkpoint_dir` | `output/repo_checkpoints/` | Per-repo CSV outputs |

### Date Filters
| Option | Default | Description |
|--------|---------|-------------|
| `date_since` | `2000-01-01` | Earliest commit date |
| `date_to` | `2030-12-31` | Latest commit date |

### Commit Filters
| Option | Default | Description |
|--------|---------|-------------|
| `only_no_merge` | `True` | Exclude merge commits |
| `only_code_files` | `True` | Only process code files |
| `only_in_branch` | `None` | Filter to specific branch |

### Commit Sampling
| Option | Default | Description |
|--------|---------|-------------|
| `commit_skip` | `10` | Take every Nth commit |
| `min_commit_interval_days` | `0` | Min days between commits |
| `min_repo_commits` | `50` | Min commits for repo |
| `min_branch_commits` | `10` | Min commits per branch |

### File Filters
| Option | Default | Description |
|--------|---------|-------------|
| `min_file_size` | `500` | Min file size (bytes) |
| `max_file_size` | `100000` | Max file size (bytes) |
| `min_lines` | `10` | Min added lines per file |
| `max_repo_size_mb` | `1000` | Max repo size (MB) |

### Content Filters
| Option | Default | Description |
|--------|---------|-------------|
| `exclude_autogenerated` | `True` | Skip auto-generated files |
| `exclude_vendored` | `True` | Skip vendored code |
| `exclude_tests` | `False` | Skip test files |
| `exclude_minified` | `True` | Skip minified files |
| `max_comment_ratio` | `0.7` | Skip files >70% comments |

### Parallelization
| Option | Default | Description |
|--------|---------|-------------|
| `num_workers` | `1` | Threads for commit processing |
| `parallel_cloning` | `True` | Clone repos in background |
| `max_cloned_repos` | `10` | Clone queue size |
| `commit_timeout` | `5` | Seconds per commit timeout |
| `batch_timeout` | `120` | Seconds per branch timeout |

| Column | Description |
|--------|-------------|
| `repo_id` | Repository ID |

| `branch` | Branch name |

| `loc_diff` | Lines of code (diff) |
| `added_lines` | Number of added lines |
| `deleted_lines` | Number of deleted lines |
| `comment_density` | Comment ratio |
| `boilerplate_score` | Boilerplate detection score |
| `cyclomatic_complexity` | Cyclomatic complexity |
| `cognitive_complexity` | Cognitive complexity |
| `token_count` | Token count |

## Troubleshooting

### Workers Hanging
If parallel workers hang on certain commits:
1. Set `num_workers=1` for sequential processing
2. Lower `commit_timeout` (e.g., 3-5 seconds)
3. GitPython issues with certain commits are logged and skipped

### No Data Being Saved
Check the debug output for filter stats:
```
[DEBUG] Repo name: X rows | no_diff=Y, not_code=Z, file_filter=W, too_few_lines=V, passed=P
```
- `too_few_lines`: Lower `min_lines` in config
- `file_filter`: Adjust size/content filters
- `not_code`: Check `only_code_files` setting

### Disk Space
Repos are cloned to `data/temp/` and deleted after processing. If deletion fails (Windows file locks), clean up manually:
```bash
rm -rf data/temp/repo_*
```
